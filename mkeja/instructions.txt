MKEJA ONBOARDING MICROSERVICE – MASTER PRODUCT PROMPT, IMPLEMENTATION INSTRUCTIONS, AND END-TO-END USER JOURNEY
PRODUCT MISSION

Design and build a production-grade, event-driven onboarding microservice for MKEJA, a digital property management and rental ecosystem. The onboarding service is responsible for onboarding landlords, property owners, property management companies, agents, properties, units, tenants, lease agreements, and tenancy relationships.

The platform must support the complete lifecycle from landlord registration to tenant move-in while maintaining strict separation between a Tenant (Person) and a Tenancy (Relationship).

This service acts as the system of record for onboarding and tenancy creation while integrating with external microservices through asynchronous events.

CORE DOMAIN PRINCIPLES
Tenant vs Tenancy

A Tenant is a person.

A Tenancy is a contractual and operational relationship between a tenant and a property unit.

A tenant can have multiple tenancies over time.

A property unit can have multiple tenancies over time.

A property unit can only have one active tenancy at any given moment.

Example:

Tenant:
John Doe

Tenancies:

Apartment A101 (2023)
Apartment B204 (2024)
Apartment C305 (Future)

The tenant remains the same person while the tenancy changes.

This distinction must be reflected throughout:

Database design
API design
Domain models
Event contracts
User interfaces
Reporting
